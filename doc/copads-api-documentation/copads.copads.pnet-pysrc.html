 27      Class to represent a Petri Net or Petri Net typed object. 
 28       
 29      The places and transition rules are represented as dictionary objects.  
 30      Places dictionary will have the name of place as key and the Place  
 31      (pnet.Place object) as value. Transition rules dictionary will have  
 32      the name of rule appended with a number (in the format of <rule name>_ 
 33      <number> in order to ensure uniqueness) as key and the value is a  
 34      dictionary to represent the transition rule (the structure of the  
 35      transition rule dictionary is dependent on the type of rules). 
 36       
 37      The following types of transition rules are allowed:  
 38          - step rule 
 39          - delay rule 
 40          - incubate rule 
 41          - ratio rule 
 42          - function rule 
 43           
 44      Step rule is to be executed at each time step. For example, if 20g of  
 45      flour is to be transferred from flour bowl to mixer bowl at each time  
 46      step, this 'add_flour' rule can be defined as: 
 47       
 48      >>> net.add_rules('add_flour', 'step',  
 49                        ['flour.flour -> mixer.flour; 20']) 
 50       
 51      A single step rule can trigger more than one token movement. For  
 52      example, the following step rule simulates the mixing of ingredients  
 53      into a flour dough: 
 54       
 55      >>> net.add_rules('blend', 'step',  
 56                        ['mixer.flour -> mixer.dough; 15', 
 57                         'mixer.water -> mixer.dough; 10', 
 58                         'mixer.sugar -> mixer.dough; 0.9', 
 59                         'mixer.yeast -> mixer.dough; 1']) 
 60                          
 61      Delay rule acts as a time delay between each token movement. For  
 62      example, the following rule simulates the transfer of 0.5g of yeast  
 63      into the mixer bowl: 
 64       
 65      >>> net.add_rules('add_yeast', 'delay',  
 66                        ['yeast.yeast -> mixer.yeast; 0.5; 10']) 
 67                         
 68      Incubate rule is a variation of delay rule. While delay rule is not  
 69      condition dependent, incubate rule starts a time delay when one or  
 70      more conditions are met. For example, 
 71       
 72      >>> net.add_rules('rise', 'incubate',  
 73                        ['10; mixer.dough -> pan.dough; \ 
 74                         mixer.flour == 0; mixer.water == 0; \ 
 75                         mixer.sugar == 0; mixer.yeast == 0']) 
 76                          
 77      sets a 10 time step delay when all flour, water, sugar, and yeast in  
 78      the mixer bowl are used up, which simulates the completemixing into a  
 79      bread dough. The 10 time step then simulates the time needed for the  
 80      dough to rise. After 10 time steps, dough in the mixer is transferred  
 81      into the pan. 
 82       
 83      Ratio rule is a variant of step rule. Instead of absolute number of  
 84      tokens to move, the movement is a percentage of the number of tokens.  
 85      For example,  
 86       
 87      >>> net.add_rules('bake', 'ratio',  
 88                        ['pan.dough -> pan.bread; 0.3; pan.dough < 1; 0']) 
 89                         
 90      will move 30% of the token value from dough in pan to bread in pan. If  
 91      the token value of dough in pan is less than 1, then the token value  
 92      of dough in pan will be set to 0. 
 93       
 94      Function rule is a generic and free-form rule, that takes the form of  
 95      a Python function. This is usually used when the transition cannot be  
 96      represented by any other rules. Given a user-defined function, FUNC,  
 97       
 98      >>> net.add_rules('cool', 'function',  
 99                        ['table.temperature -> air.temperature', FUNC,  
100                         'table.bread > 0; table.temperature > 30']) 
101                          
102      FUNC will be executed when table.bread > 0 and table.temperature > 30.  
103      The returned result of FUNC will be the token transfer from  
104      table.temperature to air.temperature.  
105       
106      FUNC takes all the places as a single parameter, such as FUNC(places),   
107      where 'places' is a dictionary with the name of each place as key.  
108      Token values in any place can be assessed. For example, 
109       
110      >>> place_names = places.keys() 
111      >>> a_place = places[places_names[0]] 
112      >>> token_set = a_place.attributes.keys() 
113      >>> a_token_value = a_place.attributes[token_set[0]] 
114      '''
